About LetsWatchTV

API's Used:

  • TVmaze - TVShow information
  • guidebox - twitter handles
  • Twitter - Fetching various pieces of information from Twitter via their API

Libraries Users:

  • Bootstrap - Frontend framework
  • lazyload.js - Lazyloading images
  • SASS - Precompiled CSS library with more power & flexibility
  • Toastr - toast notification library
  • Slick carousel - Image carousels
  • Isotope / masonry - sorting & filtering

Python

  • Flask
  • Flask Migrate
  • Alembic
  • Flask Login
  • Flask WTF (What The Form)
  • Flask SQLAlchemy
  • Requests
  • Twitter Publish
  • Werkzeug

Server Side JS

  • npm - Node's package manager for managing js dependencies
  • gulp.js - Task automator, used for SASS, JS, and livereload
  • bower - Package manager for frontend libraries

Backlog

  • Live Twitter Stream

      • Originally, Frances planned on having each tvshow have their twitter feed aggregate in real time on the tvshow details page. However, due to inability to have this possible on all tvshows it will be in the list of features to be implemented in the future when more time is allowed.

  • Visualization of Tweets/Time and Sentiment Analysis

      • Along with the live twitter stream, Frances also wanted to dive into visualizating all the the tweets that came in within a specific time period as an indicator of populatarity (assuming that the raw number of tweets is equivalent to the popularity of a show) and sentiment analysis of tweets to determine if the tvshow's rating from IMDB and RottonTomato were true. Due to the nature of machine learning being a huge topic of study, this was also pushed further back when more time is available.

    • Pending Work:

        • Currently, all of the tweets from the Summer 2016 Brazil Olympics have been saved.

      • Data Wrangling
        • Remove spam tweets
        • Remove tweets that only have retweets of twitter handles
        • Remove all tweets that are not English
  • Sharable Links

      • Allow users to share tvshows that they've favorited or are currently watching and the ability to mark a list as private or public

  • Implement typeahead/fuzzysearch
  • Watch Now!

      • Unfortunately, Netflix no longer has a public API, Hulu doesn't have a public API at all, and Apple's API is in XML there was limited time to look places that a user can stream a tvshow or purchase a tvshow (legally). It appears that other API's that have had access or still have access to the API's don't have public API's so this will be in the backlog until there is a public API.
